CCTV BACKEND EXPLANATION (Beginner-Friendly)

Location: cctv-backend/
Main role: FastAPI server that accepts video uploads, runs anomaly detectors, stores results in MongoDB, and serves analytics/alerts/streams to the frontend.

============================================================
1) OWNERSHIP LEGEND (who created each file/folder type)
============================================================
[DEV] = Developer-written source code/config.
[PKG] = Created by package/environment installation tools.
[GEN] = Auto-generated at runtime (Python cache or app output).
[ASSET] = Model/data asset used by code (not Python package code).
[TEMPLATE] = Example config template for user setup.

Important note:
- Installed Python libraries (FastAPI, torch, cv2, etc.) come from pip and live in virtual environment site-packages.
- Your project .py files inside cctv-backend are developer files, not package files.

============================================================
2) TOP-LEVEL INSIDE cctv-backend/
============================================================
.env                       [DEV] Runtime configuration values used by backend.
.env.example               [TEMPLATE] Example env file to copy/fill.
.gitignore                 [DEV] Git ignore rules for backend artifacts.
README.md                  [DEV] Backend setup and usage documentation.
requirements.txt           [DEV] List of Python dependencies to install via pip.
main.py                    [DEV] FastAPI entrypoint; mounts middleware + all API routes.
fight_model.py             [DEV] Fight model helper definitions used by fight detector.

.venv/                     [PKG] Virtual environment files created by Python/pip tools.
api/                       [DEV] API route modules.
core/                      [DEV] Shared app state/utilities (Mongo, helpers, overlays).
detectors/                 [DEV] All anomaly detector modules + registry.
models/                    [ASSET] Trained model weight files (.pth).
uploads/                   [GEN] Uploaded videos and temporary/generated media files.
__pycache__/               [GEN] Python bytecode cache files.

============================================================
3) api/ folder
============================================================
api/__init__.py            [DEV] Marks api as a Python package.
api/__pycache__/           [GEN] Python cache for api package.

api/routes/                [DEV] Route split by feature area.
api/routes/__init__.py     [DEV] Marks routes as package.
api/routes/__pycache__/    [GEN] Python cache for routes modules.

api/routes/health_routes.py     [DEV] Health check + Mongo reconnect endpoints.
api/routes/stats_routes.py      [DEV] Overview/trends stats endpoints.
api/routes/videos_routes.py     [DEV] Video detections + original/processed stream endpoints.
api/routes/alerts_routes.py     [DEV] Alerts listing/filtering + resolve/toggle endpoints.
api/routes/analytics_routes.py  [DEV] Analytics summary (trends/time patterns/hot zones).
api/routes/process_routes.py    [DEV] Upload/process endpoint with SSE event streaming.

============================================================
4) core/ folder
============================================================
core/__init__.py           [DEV] Marks core as package.
core/app_state.py          [DEV] Shared backend state/utilities:
                                - Mongo connection/init and collection helpers
                                - datetime + formatting helpers
                                - severity conversion
                                - processed-video overlay drawing and rendering helpers
                                - source video path resolution
core/__pycache__/          [GEN] Python cache for core modules.

============================================================
5) detectors/ folder
============================================================
detectors/__init__.py      [DEV] Detector registry mapping anomaly IDs to detect() functions.

Real implemented detectors:
- detectors/gunshot_detector.py  [DEV] Audio gunshot detection pipeline (ffmpeg + mel + ResNet).
- detectors/fight_detector.py    [DEV] Visual fight detection via sliding-window video model.
- detectors/fall_detector.py     [DEV] Visual fall detection using pose/LSTM classifier.

Stub/placeholder detectors (currently return empty list until model is added):
- detectors/scream_detector.py   [DEV]
- detectors/explosion_detector.py [DEV]
- detectors/crowd_detector.py    [DEV]

detectors/__pycache__/      [GEN] Python cache for detector modules.

============================================================
6) models/ folder
============================================================
models/best_fight_model.pth        [ASSET] Trained weights for fight detector.
models/gunshot_trained_model.pth   [ASSET] Trained weights for gunshot detector.
models/posture_model_best.pth      [ASSET] Trained weights for fall/posture detector.

============================================================
7) uploads/ folder
============================================================
uploads/                    [GEN] Runtime media storage created/used by API.

Contains many files like:
- <uuid>_<original_name>.mp4 / .avi  [GEN] Uploaded source videos.
- <uuid>_<original_name>_audio.wav   [GEN] Temporary extracted audio for audio detectors.
- processed/                          [GEN] Generated processed videos with overlays.

uploads/processed/ files:
- processed_<videoId>.mp4             [GEN] Rendered processed stream output.

These are not package files and not usually hand-written source files.

============================================================
8) WHICH FILES ARE FROM INSTALLATION PACKAGES vs DEVELOPER?
============================================================
From installation/package tooling:
- cctv-backend/.venv/**        [PKG]

Not package-installed (developer or project/runtime-owned):
- All .py source files under cctv-backend/api, cctv-backend/core, cctv-backend/detectors, main.py, fight_model.py  [DEV]
- requirements.txt/.env/.env.example/README.md/.gitignore  [DEV or TEMPLATE]
- models/*.pth  [ASSET]
- uploads/**, __pycache__/**  [GEN]

Short rule to remember:
- If it is your project code (.py in api/core/detectors/main), it is developer code.
- If it is in .venv/site-packages, it is installed package code.
- If it appears after running Python/app (uploads/__pycache__), it is generated.

============================================================
9) BACKEND EXECUTION FLOW (quick beginner map)
============================================================
1. main.py starts FastAPI and mounts route modules.
2. process_routes.py receives upload + selected anomalies.
3. Detector functions run (from detectors registry).
4. Events are saved to Mongo (if connected) and streamed to frontend (SSE).
5. alerts/stats/analytics/video routes read Mongo + files and return UI data.
6. videos processed stream endpoint can render overlay video using core/app_state helpers.

End of file.
