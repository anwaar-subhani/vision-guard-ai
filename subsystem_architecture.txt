ANOMALY DETECTOR SUBSYSTEM


📥 INPUT: Encoded Video Feature Vector + Encoded Audio Feature Vector
                ↓
🟧 MODULE 1: Feature Merger
   • Combines visual and audio feature vectors  
   • Aligns them on a common time scale (synchronization)
   🔹 Input: Video features + Audio features  
   🔹 Output: Unified multimodal feature set
                ↓
🟧 MODULE 2: Feature Preprocessor
   • Normalizes data (scaling, mean-variance normalization)  
   • Handles missing values or noise
   🔹 Input: Unified feature set  
   🔹 Output: Normalized feature tensor
                ↓
🟧 MODULE 3: Deep Learning Model (TensorFlow)
   • CNN or LSTM network trained on labeled data  
   • Learns spatiotemporal correlations between sound & video  
   🔹 Input: Normalized feature tensor  
   🔹 Output: Probability scores (normal / anomalous)
                ↓
🟧 MODULE 4: Decision Module
   • Applies a threshold on anomaly score  
   • Labels event as "Normal" or "Anomaly"  
   🔹 Input: Model probabilities  
   🔹 Output: Classified event label
                ↓
🟧 MODULE 5: Alert & Logging System
   • Triggers alerts (SMS, email, dashboard update)  
   • Stores anomaly records in database with timestamp & metadata  
   🔹 Input: Event label  
   🔹 Output: Alert notification / Log entry
                ↓
📤 FINAL OUTPUT: “Normal” / “Anomaly” Result + Alert Triggered


AUDIO ANALYZER SUBSYSTEM

📥 INPUT: CCTV Audio Stream / Extracted Audio Track
               ↓
🟩 MODULE 1: Audio Input Handler
   • Reads or extracts audio from video or microphone input
   • Handles sampling rate, format (WAV/MP3)
   🔹 Input: Audio signal / audio file
   🔹 Output: Raw audio waveform data
               ↓
🟩 MODULE 2: Audio Preprocessor
   • Removes background noise, normalizes amplitude
   • Converts stereo to mono, resamples if needed
   🔹 Input: Raw waveform
   🔹 Output: Clean, standardized audio signal
               ↓
🟩 MODULE 3: Feature Extractor (Librosa)
   • Uses Librosa to extract features such as:
     - MFCC (Mel-Frequency Cepstral Coefficients)
     - Spectral Centroid, Bandwidth, Zero Crossing Rate
     - Chroma and Energy
   🔹 Input: Preprocessed audio signal
   🔹 Output: Audio feature matrix (time × frequency features)
               ↓
🟩 MODULE 4: Temporal Pattern Analyzer
   • Detects unusual sound intensity patterns (e.g., sudden loud noises)
   • Tracks duration and repetition of sound spikes
   🔹 Input: Feature matrix
   🔹 Output: Temporal sound event descriptors
               ↓
🟩 MODULE 5: Feature Encoder
   • Converts extracted audio descriptors into numeric vector form
   • Normalizes and formats for AI model input
   🔹 Input: Temporal descriptors
   🔹 Output: Encoded audio feature vector
               ↓
📤 FINAL OUTPUT: Encoded Audio Feature Vector → (Sent to Anomaly Detector)




VIDEO ANALYZER SUBSYSTEM

📥 INPUT: CCTV Video Feed / Uploaded Video File
               ↓
🟩 MODULE 1: Video Input Handler
   • Reads video frames using OpenCV (cv2.VideoCapture)
   • Handles frame rate, resolution
   🔹 Input: Raw video stream
   🔹 Output: Sequence of video frames
               ↓
🟩 MODULE 2: Frame Preprocessor
   • Converts to grayscale, resizes, reduces noise
   • Prepares frames for stable processing
   🔹 Input: Raw frames
   🔹 Output: Preprocessed clean frames
               ↓
🟩 MODULE 3: Motion Detector
   • Uses frame differencing / background subtraction
   • Detects movement regions in frames
   🔹 Input: Preprocessed frames
   🔹 Output: Motion mask (highlighted moving areas)
               ↓
🟩 MODULE 4: Object Tracker
   • Tracks detected objects across frames (KCF / CSRT)
   • Assigns IDs and maintains trajectories
   🔹 Input: Motion mask + frames
   🔹 Output: Tracked object data (bounding boxes + positions)
               ↓
🟩 MODULE 5: Feature Extractor
   • Extracts visual features — posture, movement, object shape, speed
   🔹 Input: Tracked object data
   🔹 Output: Raw visual features
               ↓
🟩 MODULE 6: Feature Encoder
   • Converts extracted features into numeric format (vector)
   • Normalizes and prepares input for AI model
   🔹 Input: Visual features
   🔹 Output: Encoded feature vector
               ↓
📤 FINAL OUTPUT: Encoded Visual Feature Vector → (Sent to Anomaly Detector)
