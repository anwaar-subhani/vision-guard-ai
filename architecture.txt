Layered Architecture Explanation — Video Surveillance Anomaly Detector
🟦 1. Presentation Layer (React 💻)
🎯 Purpose:

Acts as the user interface — the part visible to the end-user (security staff, admin, or operator).

🧩 Modules:
Module	Description
Dashboard UI	Displays real-time camera feeds and highlights detected anomalies.
User Controls	Lets the user select CCTV feed, adjust sensitivity, or acknowledge alerts.
Notifications Panel	Shows alerts or anomaly logs coming from backend layers.
🔄 Interaction:

Sends user requests (e.g., “start detection,” “view report”) to the Application Layer.

Receives results and alert data from the Application Layer to display on screen.

🟩 2. Application Layer (Python ⚙️)
🎯 Purpose:

This layer is the controller that manages communication between the UI and the AI modules.
It translates user commands into backend operations and manages system workflow.

🧩 Modules:
Module	Description
Stream Manager	Handles video stream input — starts, pauses, or ends video feed processing.
Event Handler	Responds to events (like “motion detected” or “AI anomaly detected”).
Alert Manager	Collects anomaly alerts from Business Layer and sends them to the UI.
🔄 Interaction:

Receives commands from Presentation Layer.

Sends video/audio data to Business Layer for analysis.

Gets detection results back and forwards them to UI.

🟨 3. Business Layer (AI Processing 🧠)
🎯 Purpose:

This is the core intelligence of the system — it processes raw data to detect abnormal behavior or sounds using AI.

🧩 Modules:
Module	Description
Video Analyzer (OpenCV 🎥)	Extracts frames and identifies motion or unusual visual patterns.
Audio Analyzer (Librosa 🎵)	Analyzes sounds for loud, sudden, or unusual noise patterns.
Anomaly Detector (TensorFlow 🔶)	Runs the trained AI model (CNN or other ML model) to classify whether the activity is normal or abnormal.
🔄 Interaction:

Receives video/audio data from Application Layer.

Processes it using AI algorithms.

Sends anomaly detection results to Application Layer.

Fetches or stores model data through Persistence Layer if needed (e.g., to load trained models).

🟥 4. Persistence Layer (Data Access Layer 🐍)
🎯 Purpose:

Handles data access operations — a middle layer between AI logic and raw data storage.
This layer ensures data consistency, security, and reusability.

🧩 Modules:
DAO	Description
AlertDAO	Manages saving and retrieving alert logs (e.g., time, type, location).
ModelDAO	Loads or updates trained AI models from the repository.
DatasetDAO	Handles training/test data storage or retrieval for retraining.
🔄 Interaction:

Called by Business Layer when model or data access is needed.

Sends or retrieves data from the Data Layer.

🟧 5. Data Layer (Storage 🗄️)
🎯 Purpose:

This is the lowest layer — it stores all persistent information such as datasets, models, and logs.

🧩 Components:
Component	Description
Dataset Storage 📁	Holds raw CCTV video/audio datasets for training or retraining models.
Model Repository 🔶	Contains AI models (e.g., TensorFlow .h5 files).
Alert Logs DB 🐘	Database for storing alert history, timestamps, and metadata.
🔄 Interaction:

Accessed by the Persistence Layer through DAO classes.

Provides long-term storage and retrieval support.

🔁 Layer Interactions (Overall Flow)
User (UI) → Application Layer → Business Layer → Persistence Layer → Data Layer
                                    ↑                ↓
                                   Alerts ←-------- Data Access

Step-by-step Example:

🧑‍💻 User starts monitoring → (Presentation Layer → Application Layer)

🎥 Stream Manager captures video and sends frames → (Application → Business)

🧠 Business Layer analyzes frames/sounds using TensorFlow, OpenCV, Librosa

🚨 Anomaly Detected! → Business Layer sends alert to Application Layer

🐍 Persistence Layer logs alert and stores in DB

💾 Data Layer saves record

💻 UI shows real-time alert to user

🎨 Why This Layered Architecture is Best

✅ Modular — Each layer can be updated independently.
✅ Scalable — You can replace TensorFlow with another AI framework later.
✅ Maintainable — DAO separates logic from data access.
✅ Clear Communication Flow — UI ↔ Controller ↔ AI ↔ Database.
✅ Extensible — New modules (like face recognition or object counting) can be added easily.
